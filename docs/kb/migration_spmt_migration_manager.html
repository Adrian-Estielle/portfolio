SPMT + Migration Manager (agents, templates, logs)

Practical approach for Microsoft migration tooling: scan first, template-driven mapping, pilots, log-driven remediation, then scale.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

Goal

Run Microsoft-native migration tooling in a way that’s operationally predictable: service accounts, templates, logs, and repeatable batches.

SharePoint Migration Tool (SPMT)

  1. Install SPMT on a dedicated migration workstation.
  2. Sign in with an approved migration/service account.
  3. Run a scan first (don’t blind-run): identify path length, invalid characters, locked files.
  4. Use CSV templates for repeatability (source → destination mapping).
  5. Pilot → then expand waves.

Migration Manager (agent-based)

  1. Install agents on servers close to the data.
  2. Register agents to the tenant with approved account.
  3. Download and complete mapping templates (shares → SharePoint sites/libraries).
  4. Run pilots; review logs; remediate failures before scaling.

Reading reports and logs

  • Failures by category: invalid name, path length, access denied, locked files.
  • Skipped vs failed (skipped may be expected; failed must be fixed).
  • Delta behavior: confirm subsequent runs are smaller and converging to zero changes before cutover.
