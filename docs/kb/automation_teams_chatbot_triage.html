Teams chatbot triage (driver reset + auto ticket documentation)

Automated remediation + auto-documented resolution to reduce repetitive tickets while preserving an audit trail.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

Use case

Common post-dock / post-home-office display issues should not consume support capacity. Automate detection + remediation, but still document it.

Behavior

  • Collect quick diagnostics (GPU/driver versions, known device IDs, recent changes).
  • Use the same detection signals Windows “Troubleshoot” often surfaces for driver-related issues.
  • Remove/reinstall known-problem drivers with a safe rollback path.
  • Always create a Zendesk ticket for documentation; if the fix works, the ticket is auto-resolved with remediation notes.
  • Escalate only if the scripted fix doesn’t resolve.
