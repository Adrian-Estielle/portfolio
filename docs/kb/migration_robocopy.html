Robocopy pre-seed + delta sync (and log review)

Repeatable robocopy pattern for staged migrations, including what to look for in logs and how to remediate common failures.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

Use case

Pre-seed bulk data early, then run deltas leading up to cutover. This reduces the cutover window and lowers risk.

Starter command (dry-run first)

Warning: /MIR can delete on destination. Use /L (list-only) first.
robocopy "D:\SourceShare" "E:\StagingShare" /MIR /ZB /R:2 /W:5 /MT:32 /DCOPY:DAT /COPY:DATSO 
  /NP /TEE /LOG+:"C:\Logs\robocopy-seed.log"

How I review robocopy logs

  1. Check summary totals at bottom (copied/failed/skipped).
  2. Search for ERROR and 0x codes.
  3. Validate “Skipped” vs “Failed” (failed requires remediation).

Common failures

  • Path length / invalid names → restructure or normalize before migration.
  • Access denied → fix permissions / inheritance / service account rights.
  • Locked files → schedule + coordinate with app owners.
