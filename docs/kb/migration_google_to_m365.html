Google Workspace → Microsoft 365 migration (high-level)

A practical wave approach for mail/calendar/files migrations with validation and controlled cutovers.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

Scope

Mail, calendar, and files are separate workstreams with different failure modes. Treat them as coordinated waves, not one “big move.”

Wave plan (high-level)

  1. Pilot batch with real workflows.
  2. Identity readiness (domains, sign-in policies, MFA readiness, support path).
  3. Mail migration strategy (staged where possible; validate clients).
  4. Calendar migration validation (shared calendars, rooms/resources).
  5. Drive/files mapping (ownership + sharing model mapped intentionally).
  6. Cutover + verification + support readiness.
