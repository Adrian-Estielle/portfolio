Offboarding / termination automation

Fast revoke + approved preservation (manager redirect, delegation) + deferred deletion with approval gates.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

Goal

Fast access revocation with approved preservation steps (mailbox retention/delegation) and controlled irreversible actions.

Termination flow (sanitized)

  1. Disable sign-in + revoke sessions.
  2. Mailbox handling (authorized): auto-reply directs senders to the terminated user’s manager; delegation/access only where approved; capture mailbox state.
  3. Strip groups/roles; move user to terminated OU/container.
  4. Timer-based cleanup: after N days, delete user (with approvals for destructive steps).
  5. Optional: Graph API cleanup for MFA factors (approval gated).

Why approvals matter

Clearing MFA factors / deleting accounts is irreversible. Gate it behind explicit approval and log the action.
