Onboarding automation: minimal inputs → user + access baseline

  • Inputs: first/last, title, office, manager.
  • Generate UPN/email from convention; collision checks.
  • Assign security groups via a role/region matrix (replaces “copy another user”).
  • Licensing rule example (real-world): base license (ex: E5) auto-assigns when Office Location + Email fields are populated; other group/license logic keys off those attributes.
  • Outputs: summary + logs for audit and replay.

Offboarding / termination automation: revoke access fast + preserve required data

  • Disable sign-in + revoke sessions.
  • Mailbox handling (authorized): auto-reply that directs senders to the terminated user’s manager; delegate access where approved; capture mailbox state.
  • Strip groups/roles; move to terminated OU/container.
  • Deferred deletion with approvals for irreversible steps (ex: MFA factor clears via Graph API).

Mapped drive “self-heal” automation

  • Detect known-bad mappings; remove mapping; clear stale credentials; re-map; log.
  • If it keeps coming back: audit the underlying permissions (ACL drift) and fix root cause instead of re-mapping forever.

Teams chatbot triage: dock/home-office display issues (driver reset)

Intent: eliminate “simple-but-common” tickets while still keeping documentation.

  • Bot gathers quick diagnostics (GPU/driver versions, dock model, monitor chain, recent changes).
  • Uses the same types of detection signals Windows “Troubleshoot” surfaces (Device Manager status/error codes, basic driver health checks) to categorize likely issues.
  • Automates remove/reinstall for known-problem drivers with a safe rollback path.
  • Still creates a Zendesk ticket (for audit/history) — and if the scripted fix works, it auto-documents the resolution and closes the ticket.
  • Escalates only if the scripted fix fails or diagnostics indicate hardware fault.
