Export on-prem permissions → recreate cleanly in Microsoft 365

Repeatable approach to mapping ACL intent into Entra groups + SharePoint roles, with validation before scale.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

Goal

Export legacy ACL intent (groups + access), then recreate cleanly in M365 using Entra groups and SharePoint roles — without carrying chaos forward.

High-level steps

  1. Export folder ACLs + group membership (CSV/JSON) from the file share.
  2. Normalize: flatten nested group sprawl; identify exceptions; reduce deny entries where possible.
  3. Create Entra security groups representing access units (owners/members/visitors patterns).
  4. Assign groups to sites/libraries with consistent roles.
  5. Validate with real user contexts before scaling.

SMB reminder: Share vs NTFS

Effective network access is the most restrictive combination of Share + NTFS. Share does not override NTFS — it can only further restrict.

Sanitized export skeleton (pattern)

# Export ACLs (pattern)
	Get-ChildItem "<SHARE_ROOT>" -Directory -Recurse |
	  ForEach-Object {
    $acl = Get-Acl $_.FullName
    [pscustomobject]@{
      Path   = $_.FullName
      Access = ($acl.Access | ForEach-Object { "$($_.IdentityReference):$($_.FileSystemRights)" }) -join "; "
    }
  } | Export-Csv ".\acl-export.csv" -NoTypeInformation
