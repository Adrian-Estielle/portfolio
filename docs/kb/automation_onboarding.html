Onboarding automation (role/region matrix + licensing rules)

Minimal inputs → consistent identity, access baseline, and licensing, with logs for audit/replay.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

What this solves

Reduce onboarding to minimal inputs, then apply a consistent access/licensing baseline using a role/region matrix instead of “copy some other user and hope it’s right.”

Inputs

  • First name, last name
  • Title / role
  • Office / region
  • Manager

Provisioning flow

  1. Generate UPN/email from convention; collision check.
  2. Create user + populate attributes (incl. manager).
  3. Assign security groups from a standardized matrix (role/region mapping).
  4. License automation: base license (ex: E5) assigned automatically when key attributes are present (Office Location + Email), with other group/license rules layered on top.
  5. Output summary log (audit-friendly, replayable).
