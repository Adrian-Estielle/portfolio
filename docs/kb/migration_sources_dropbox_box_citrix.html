Dropbox / Box / Citrix-style sources — migration notes

What changes when the source isn’t a classic SMB share: ownership, links, permissions, and client behavior.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

Why this needs its own page

Dropbox/Box/Citrix-style sources often differ from SMB shares in ownership models, link behavior, and permission semantics. Plan accordingly.

Practical approach

  1. Inventory: volume, sharing/link sprawl, ownership patterns, business workflows.
  2. Pick the migration path (native tooling where it fits; third-party where required).
  3. Pilot with a set that contains the worst edge cases.
  4. Validate: permissions, links, sync behavior, and user experience.
  5. Wave cutover: low dependency → high dependency.

Common gotchas

  • Link remediation doesn’t scale manually.
  • Owner vs contributor expectations must map cleanly into SharePoint roles.
  • Sync thresholds and “too many items” behavior show up late if you pilot wrong.
