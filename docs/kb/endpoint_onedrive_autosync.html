Auto-sync SharePoint libraries to File Explorer (Intune + OneDrive)

Group-targeted library auto-mount to reduce repeat tickets and eliminate manual sync steps.
Sanitized note: repeatable runbook format. No tenant IDs, secrets, internal hostnames, customer identifiers.

Use case

When “sync this SharePoint library” becomes recurring support cost (especially for a heavy-use department), you can auto-mount it via Intune + OneDrive policy.

The part guides gloss over

  • The library string you copy is often URL-encoded.
  • Convert it before deploying in policy.
[uri]::UnescapeDataString("tenantId%3d...%26siteId%3d...%26webId%3d...%26listId%3d...%26webUrl%3d...")

Scaling pattern

  • Use group membership as the source of truth: membership = permission + auto-mount intent.
  • Keep a mapping table: Library → Group → ID string → owner.
  • Script retrieval of site/list identifiers when managing many libraries.

Validation

  1. Add a test user to the group.
  2. Sign in and confirm OneDrive is running.
  3. Confirm the library mounts in File Explorer and points to the correct site/library.
