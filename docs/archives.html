Project Archives (sanitized)

This is where sanitized scripts, logs, templates, and legacy documentation get published as supporting evidence.

Scripts archive

Place sanitized scripts under docs/archive/scripts/ and link them from KB pages.
  • Onboarding/offboarding automation snippets (sanitized)
  • Drive mapping self-heal scripts
  • Migration helper scripts (permissions export, log parsers, etc.)

Documentation + templates archive

Place sanitized docs/templates under docs/archive/docs/ and docs/archive/templates/.
  • Migration CSV templates (sanitized)
  • Runbook PDFs or internal KB exports (sanitized)
  • Example log excerpts (sanitized) under docs/archive/logs/
Sanitization rule: no tenant IDs, internal domains, user identifiers, private URLs, or secrets.
