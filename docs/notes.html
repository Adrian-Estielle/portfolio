Endpoint notes*

Real-world patterns from Intune/Modern Workplace work: provisioning, compliance triage, and “reduce ticket load” automations.

Autopilot / enrollment reliability

  • Pre-flight: verify MFA readiness, stable network during OOBE, firmware baseline (TPM/Secure Boot), correct profile assignment and naming/group tags.
  • Failure mode: first sign-in before MFA setup can create inconsistent states. My fix path is cleanup + re-enroll (don’t waste hours on a zombie object).
  • Failure mode: network drops during OOBE. If it’s already broken, wipe/re-run with known-good connectivity (hardwired when possible).
  • OOBE troubleshooting: validate network, capture logs, and use a repeatable remediation sequence (kept sanitized in public docs).

Compliance triage (“Not compliant”)

  • Start with staleness: last check-in, policy evaluation time, user sign-in state.
  • Identify the exact failing control (encryption, OS version, Defender, firewall, jailbreak/root, required apps, etc.).
  • Choose the right lever: user-fixable vs re-enroll vs wipe (fast resolution beats heroic guessing).

SharePoint library auto-sync to File Explorer (ticket reduction)

  • Use scripting to gather required SharePoint identifiers (site/library IDs) needed for OneDrive auto-mount configuration.
  • Use Intune targeting (group-based assignment) to ensure only the right users get the right library sync.
  • Include a remediation path when OneDrive gets stuck (reset client/cache, re-apply policy, verify auth/token health).

OEM bloatware baseline (stability)

  • Maintain a short “known removals” list by vendor; remove early during provisioning.
  • Validate Office/Excel behavior, performance, and driver stability after removal.

Automation notes*

PowerShell-first automation patterns (AD/Azure/M365/Graph), plus practical support tooling.

User lifecycle automation (onboarding)

  • Start with required inputs: name, title, office/region, manager.
  • Generate username/email by convention.
  • Apply default groups using a title/region matrix (stop depending on messy “copy a random user” patterns).
  • Evolve into a GUI for service desk workflows (create / LOA disable / LOA enable / terminate).

Termination / offboarding automation

  • Immediate lockout: disable sign-in + revoke sessions.
  • Mailbox: hold/retention as required, autoreply to manager, delegate mailbox access.
  • Strip entitlements: group membership + privileged roles; move to Terminated OU; staged deletion timer.
  • Use an approval checkpoint before destructive actions (MFA method cleanup, final deletion).

Day-2 support automations

  • Mapped drives not reconnecting: remove stale mappings + re-map cleanly + credential cleanup.
  • Self-heal scripts for repeat offenders (reduce repeat tickets).

CI as proof (portfolio)

  • Not “I know CI” — the portfolio shows successful pipeline execution + artifacts + screenshots.

Migration notes*

Cutover planning that prioritizes reliability: pre-seed early, delta often, validate always.

Pre-seed + delta strategy

  • Copy data early (Migration Manager / SharePoint tools / robocopy / AzCopy depending on source/target).
  • Run periodic delta syncs so cutover day is small.
  • Freeze change window → final delta → validate → communicate rollback plan.

Spreadsheet link repair (real-world pain)

  • Accept old + new locations as parameters, update only links that should change.
  • Use dev/test copies first (never experiment on live finance data).
  • Handle hidden sheets safely (unhide → repair → rehide).
  • Preserve last-modified timestamps when required for audit/history.

Cheat sheet*

  • RBAC vs sharing vs NTFS/SMB: know which control plane you’re in before troubleshooting.
  • Permission sprawl: breaking inheritance too deep creates long-term operational debt.
  • Group-based access beats direct user perms: keeps offboarding sane.
  • Endpoint triage order: staleness → policy failure → fix lever (don’t guess first).
