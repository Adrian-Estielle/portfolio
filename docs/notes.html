KB: AD onboarding automation pattern (sanitized)

A safe pattern I use for Directory Services onboarding/maintenance: export baseline ‚Üí run scoped changes (OU-first) ‚Üí export after ‚Üí validate edge cases.
  • Safety rails: dry-run by default, explicit apply switch, and -WhatIf/-Confirm support.
  • Validation: before/after CSV exports and a small test-user harness to catch false positives.
  • Common edge cases: leave-of-absence markers, vendor/temp/test accounts, and OU subtree scope surprises.
